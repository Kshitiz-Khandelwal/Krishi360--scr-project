# Krishi360 - Project Context & Memory

This document serves as the **core memory and context guide** for developers and AI agents interacting with the Krishi360 codebase. It outlines the project's purpose, architecture, features, and future goals, ensuring everyone (human or AI) understands the "big picture."

## 1. Project Overview
**Krishi360** is a comprehensive, full-stack farming web application designed to assist farmers. It provides localized crop recommendations, tracks farming progress, fetches crop prices, alerts on weather/diseases, and integrates an AI chatbot for agricultural advice.

## 2. Tech Stack & Architecture
- **Frontend:** React + Vite, styled using Tailwind CSS (`app` or `src` directory depending on Vite config).
- **Backend:** [Convex](https://convex.dev/) (used for real-time database, server functions, and scheduled cron jobs).
- **Authentication:** Convex Auth (currently configured for anonymous auth or custom setup in `convex/auth.ts`).
- **AI Integration:** Integration with a local LLM (Ollama utilizing `tinyllama`, or similar models) for answering farmer questions and providing recommendations (see `.test_ollama.js`).

## 3. Database Schema (Convex DB)
The core knowledge representation of the application is maintained in `convex/schema.ts` and includes the following tables:
- `farmers`: Stores user profiles including location, farm size, soil type/pH, and local climate data.
- `crops`: Master data for crops, including climate/soil requirements, market price, growth days, water requirements, and disease risk.
- `recommendations`: ML/Data-driven recommendations matching farmers to specific crops (calculates a Crop Suitability Index - CSI).
- `farmProgress`: Tracks the lifecycle of crops currently planted by farmers, including Growing Degree Days (GDD), watering schedules, and notes.
- `weatherAlerts`: Stores dynamic location-based alerts regarding rain, temperature spikes, or pest/disease outbreaks.
- `chatHistory`: Logs farmer interactions with the integrated AI chatbot.
- `pastCrops`: Historical data on previously harvested crops, yield quantities, revenue, and buyer information.

## 4. Key Workflows
- **Farmer Onboarding:** A farmer creates an account and sets their land context (soil, location, size).
- **Crop Planning:** The app recommends crops based on the farmer's soil characteristics and current seasons/prices.
- **Monitoring:** As crops grow, the farmer logs details. The app provides watering schedules and alerts utilizing GDD calculations.
- **AI Consultation:** Farmers can ask localized agricultural questions through the UI, which calls out to the integrated LLM (handled in `chatbot.ts`).

## 5. Agent / AI Assistant Guidelines
When assisting with this codebase, please adhere to:
1. **Convex Best Practices:** Server functions run securely in Convex. Keep HTTP routes in `convex/http.ts` separated from typical `query` and `mutation` handlers. 
2. **Environment & Security:** Any new AI API keys or connection URLs (like Ollama endpoints) should be strictly kept in environment variables and fetched via `process.env` on the Convex backend.
3. **Database Relationships:** Many tables depend heavily on `farmerId` and `cropId`. Be mindful of indexes (e.g., `.index("by_farmer", ["farmerId"])`) when creating new data access logic.
4. **Typesafety:** All Convex API returns and arguments must use Convex validator functions (`v.string()`, `v.number()`, `v.object()` etc.) to ensure robust types.

## 6. Current State & Known Scripts
- To spin up the stack: `npm run dev` (starts both Vite and Convex).
- Test local AI endpoint: `node .test_ollama.js` runs a check against the local `11434` port API.
